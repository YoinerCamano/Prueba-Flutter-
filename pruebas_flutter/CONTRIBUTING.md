# Guía de Contribución

## Flujo de trabajo
- Rama principal: `main`
- Features en ramas `feature/...`
- Pull Requests con descripción y pruebas manuales

## Estilo
- Dart: usar `flutter format` y buenas prácticas BLoC
- Commits: mensajes claros y en español

## Revisiones
- Requerir 1 aprobación para fusionar PRs
- Ejecutar `flutter analyze` antes de enviar
